# Common AWS CLF-C02 Mistakes

Many students lose marks not because they do not know AWS services, but because they misunderstand what the question is actually asking.

This document contains the most common mistakes made during the AWS Certified Cloud Practitioner (CLF-C02) exam.

---

# Mistake 1 — Memorizing Instead Of Understanding

Many students try to memorize answers from dumps.

This works only when the exact same question appears.

AWS frequently changes:

- Question wording
- Scenarios
- Options

Concepts remain the same.

Questions change.

Always learn:

- What service is being tested
- Why the answer is correct
- Why other options are wrong

---

# Mistake 2 — Ignoring Keywords

AWS questions usually contain keywords that directly point to the correct service.

Examples:

| Keyword | Service |
|----------|----------|
| Monitoring | CloudWatch |
| Audit Logs | CloudTrail |
| Threat Detection | GuardDuty |
| Vulnerability Scan | Inspector |
| DNS | Route 53 |
| CDN | CloudFront |
| NoSQL | DynamoDB |
| Data Warehouse | Redshift |

Always identify keywords before reading options.

---

# Mistake 3 — Confusing Similar Services

Common examples:

- CloudTrail vs CloudWatch
- SNS vs SQS
- GuardDuty vs Inspector
- WAF vs Shield
- EBS vs EFS vs S3
- DynamoDB vs Redshift

Study service comparisons carefully.

---

# Mistake 4 — Reading Too Fast

Many questions contain words like:

- MOST cost-effective
- BEST solution
- LEAST operational effort
- HIGHEST availability

These words change the answer completely.

Read every question carefully.

---

# Mistake 5 — Ignoring The Wrong Answers

Wrong options are learning opportunities.

Ask:

- What does this service actually do?
- Why is it wrong here?
- Where would it be correct?

This helps eliminate options faster during the exam.

---

# Mistake 6 — Overthinking

Many AWS service names describe their purpose.

Examples:

- CloudWatch → Monitoring
- Route 53 → Routing
- Storage Gateway → Storage Bridge
- Security Hub → Security Dashboard
- Auto Scaling → Automatic Scaling

Use common sense whenever possible.

---

# Mistake 7 — Skipping Mock Tests

Mock tests reveal:

- Weak domains
- Confusing services
- Repeated mistakes

Treat mock tests as learning tools, not scorecards.

---

# Mistake 8 — Learning New Topics Before The Exam

Last-minute learning creates confusion.

Final revision should focus on:

- Comparisons
- Keywords
- Exam traps
- Service mappings

Not entirely new services.

---

# Final Advice

The AWS exam rewards:

- Concept recognition
- Service identification
- Elimination skills

More than memorization.

Understand first.

Memorize later.
